# Zabbix template Huawei AC6508
**Появилась необходимость мониторинга конкретного SSID на контроллере доступа Huawei AC6508, но столкнулся с проблемой, что название каждой SSID зашифрован в OID в формате ASCII. Пришлось написать скрипт, который будет из ASCII конвертировать имя в формат понятный человеку - то есть в буковки.**

Шаблон работает с помощью подставления аргументов, но при использовании SNMPv2 и SNMPv3 количество аргументов разное, поэтому я создал для каждой версии по шаблону. Аргументы будем подставлять в макросах узла.

## Инструкция:

- Импортировать шаблоны в Zabbix.
- Скрипт ascii_decod.py положить на сервер заббикса в папку /usr/lib/zabbix/externalscripts, либо на сервере заббикс прокси, если узел будет мониториться через прокси сервер.
- В настройках <ins>/etc/zabbix/zabbix_proxy.conf</ins> раскомментировать или добавить строчку **ExternalScripts=/usr/lib/zabbix/externalscripts**.
- Дать скрипту права <ins>chmod +x /usr/lib/zabbix/externalscripts/ascii_decod.py</ins>**

### Для работы шаблона в узле должны быть прописаны следующие макросы:

**Для SNMPv2:**
- **{$IP_ADDR}** - ip адрес
- **{$SNMP_COMM}** - community, который настроен на сервере
- **{$OID}** - OID, через который скрипт будет получать список имен точек доступа и конвертировать их в читаемые имена. Например, 	<ins>1.3.6.1.4.1.2011.6.139.17.1.2.1.16</ins>

**Для SNMPv3:**
- **{$IP_ADDR}** - ip адрес
- **{$SNMP_COMM}** - community, который настроен на сервере
- **{$OID}** - OID, через который скрипт будет получать список имен точек доступа и конвертировать их в читаемые имена. Например, 	<ins>1.3.6.1.4.1.2011.6.139.17.1.2.1.16</ins>
- **{$NAME}** - имя пользователя SNMPv3, который указан на сервере
- **{$LEVEL}** - уровень (authPriv)
- **{$AUTH_PROTO}** - протокол аутентификации (например, SHA-256)
- **{$ENCYPRO_PROTO}** - протокол шифрования (например, AES)
- **{$AUTH_PASS}** - пароль аутентификации
- **{$ENCYPRO_PASS}** - пароль шифрования
